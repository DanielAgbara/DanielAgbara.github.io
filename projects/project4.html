Scenes

  • Earth Maze: Standard terrain to ease players into core mechanics.
  • Ice Maze: Slippery surfaces challenge navigation with momentum-based movement.
  • Mud Maze: Movement is slowed, simulating heavy drag and encouraging tactical pacing.

Player

I implemented a responsive third-person player controller that included:

  • Smooth locomotion and jumping
  • Dual attack styles triggered via animation events
  • Block and damage reactions
  • Modular health system with UI elements that update in world space
  • Integration of Cinemachine FreeLook for fluid and reactive camera movement

Enemies

The enemy AI was designed using a Finite State Machine (FSM), and included:

  • Patrol behavior using Unityâ€™s NavMesh
  • Chase and detect logic using raycasts
  • Alternating melee attack patterns
  • Stuck-state detection and recovery
  • Full death sequence with locked-out states and synchronized animations
  • Three visually distinct elemental enemy models for visual and gameplay variation

Tools Used

  • Unity Engine (2022.x)
  • C# for gameplay scripting
  • Cinemachine for third-person camera control
  • NavMesh for pathfinding and movement
  • GitHub for version control and team collaboration
  • Blender (assets and rigs), Mixamo (animations)
